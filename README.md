# Assignment 3 — FastAPI Text Chunking & AI Utilities

A FastAPI application that splits large text into chunks using LangChain's
`RecursiveCharacterTextSplitter`, plus three bonus features: file upload
(CSV/PDF → text), an LLM Q&A endpoint (Groq), and an AI-generated Manim
animation endpoint (Gemini + Manim).

## Project Structure

```
.
├── main.py               # FastAPI app — all endpoints
├── chunk.py              # Text chunking logic (RecursiveCharacterTextSplitter)
├── file_reader.py        # CSV/PDF → text extraction
├── llm_query.py           # Groq LLM query logic
├── manim_generator.py     # Gemini → Manim code → rendered video pipeline
├── requirements.txt        # Python dependencies
├── Dockerfile               # For Hugging Face Spaces / Render deployment
├── .env.example             # Template for required API keys
└── .gitignore
```

## Features

| Endpoint | Method | Description |
|---|---|---|
| `/` | GET | Welcome message |
| `/chunk-text` | POST | **(Required)** Splits input text into chunks |
| `/upload-file` | POST | **(Bonus)** Upload a `.csv` or `.pdf`, get text back |
| `/ask-llm` | POST | **(Bonus)** Ask a question, get an answer from Groq |
| `/generate-animation` | POST | **(Bonus)** Describe an idea, get a Manim-rendered `.mp4` |

Interactive API docs are auto-generated by FastAPI at `/docs` once the
server is running.

## Setup (Local)

### 1. Clone the repo and create a virtual environment

```bash
git clone <your-repo-url>
cd <your-repo-name>
python3 -m venv venv
source venv/bin/activate        # On Windows: venv\Scripts\activate
```

### 2. Install system dependencies (required for Manim)

Manim needs `ffmpeg` and Cairo/Pango development libraries to render videos.

**On Ubuntu/Debian/WSL:**
```bash
sudo apt-get update
sudo apt-get install -y ffmpeg libpango1.0-dev libcairo2-dev pkg-config build-essential python3-dev
```

**On macOS (using Homebrew):**
```bash
brew install ffmpeg pango cairo pkg-config
```

**On Windows:**
Install [ffmpeg](https://ffmpeg.org/download.html) and follow Manim's
[official Windows installation guide](https://docs.manim.community/en/stable/installation/windows.html)
— it's the trickiest platform for Manim, so the official guide is the most
reliable.

If you only want the **required** features (chunking, no animation), you
can skip this step entirely and remove `manim` from `requirements.txt`.

### 3. Install Python dependencies

```bash
pip install -r requirements.txt
```

### 4. Set up your API keys

```bash
cp .env.example .env
```

Open `.env` and fill in:
- `GROQ_API_KEY` — get a free key at https://console.groq.com/keys
- `GEMINI_API_KEY` — get a free key at https://aistudio.google.com/apikey

(Skip whichever bonus feature you're not using — the rest of the app
works fine without it; only that specific endpoint will return an error.)

### 5. Run the server

```bash
uvicorn main:app --reload
```

Visit **http://127.0.0.1:8000/docs** to test every endpoint interactively.

## Example Usage

### Chunk text
```bash
curl -X POST http://127.0.0.1:8000/chunk-text \
  -H "Content-Type: application/json" \
  -d '{"text": "Your long text here...", "chunk_size": 50, "chunk_overlap": 10}'
```

### Upload a file
```bash
curl -X POST http://127.0.0.1:8000/upload-file -F "file=@yourfile.pdf"
```

### Ask the LLM
```bash
curl -X POST http://127.0.0.1:8000/ask-llm \
  -H "Content-Type: application/json" \
  -d '{"query": "What is FastAPI?"}'
```

### Generate an animation
```bash
curl -X POST http://127.0.0.1:8000/generate-animation \
  -H "Content-Type: application/json" \
  -d '{"idea": "a circle morphing into a square"}' \
  --output animation.mp4
```
This can take 15–60+ seconds depending on Gemini's response time and
animation complexity.

## Deployment

### Hugging Face Spaces (recommended — Docker handles system deps automatically)
1. Create a new Space at https://huggingface.co/new-space with SDK = **Docker**.
2. Push this repo's files (including the `Dockerfile`) to the Space's git repo.
3. In the Space's **Settings → Repository secrets**, add `GROQ_API_KEY` and
   `GEMINI_API_KEY`.
4. The Space will build and start automatically.

### Render
1. Create a new **Web Service** at https://dashboard.render.com/, connect
   your GitHub repo.
2. Choose **Docker** as the environment (so the `Dockerfile`'s system
   dependencies for Manim are installed automatically).
3. Add `GROQ_API_KEY` and `GEMINI_API_KEY` under **Environment**.
4. Deploy.

## Notes

- `chunk_overlap` must always be smaller than `chunk_size`, or the API
  returns a `400` error explaining why.
- The Manim feature requires significantly more memory/CPU than a typical
  free-tier deployment instance provides for video encoding — if renders
  time out on a free deployment tier, that's a platform resource limit,
  not a code bug.
- API keys are never committed to this repo — only `.env.example` is
  tracked; your real `.env` is git-ignored.
